# Протокол «технического предела» запоминания

Доказательный протокол для подготовки к собеседованию через Anki2. Все числа
проверены отдельным агентом-скептиком; где источник слабый — помечено.
Структура — вертикальная: **принцип → как применить в Anki2 → источник**.

---

## I. Как писать карточки (кодирование — первый контакт)

| Принцип | Как применить | Доказательство |
|---|---|---|
| **Активное припоминание** | Любой факт → вопрос-карточка; отвечай ДО переворота. «Не вспомнил» = честный Again | Roediger & Karpicke 2006: +21 п.п. через неделю vs перечитывание ✅ |
| **Атомарность (1 карта = 1 факт)** | Не «расскажи про замыкания», а серия точечных вопросов. Композитную карту топит самый слабый кусок | Wozniak «20 rules»; предел рабочей памяти 4±1 (Cowan 2001) ✅ |
| **Generation effect** | Переписывай ответ СВОИМИ словами и СВОИМ кодом, не копируй доку. Чужие колоды — только источник вопросов | Bertsch 2007, d=0.40 (445 эффектов) ✅ |
| **Вопрос «почему?»** | К каждому факту — карта-«почему» (на Senior спрашивают именно это) | Pressley 1987; Dunlosky 2013 ✅ |
| **Метод Фейнмана** | Карта не создаётся, пока не можешь объяснить факт джуну простыми словами | Chi 1994; Fiorella & Mayer 2013 ✅ |
| **Pretesting** | Перед чтением темы прогони колоду «вслепую» — даже неверные попытки усиливают последующее кодирование | Kornell, Hays & Bjork 2009 ✅ |
| **Иерархия (вертикальная структура)** | Теги `js::async::event-loop`; в начало колоды — «карты-карты» («из каких 4 частей состоит реактивность Vue?») | Bower 1969: организованное ≈ 3× лучше (65% vs 18%) ✅ |
| **Cloze-пропуски** | Для синтаксиса/сигнатур: `watch(src, cb, { ___: true })` → `deep`. Быстрейшая конвертация в карты | generation effect (перенос d≈0.40) ⚠️ |
| **Dual coding** | К системному дизайну/event loop — своя нарисованная схема | Paivio; Mayer 2009 ✅ |
| **Мнемоника точечно** | SOLID/ACID/RAIL и свои акронимы — только для упрямых списков, как костыль извлечения | Dunlosky 2013: keyword «low utility» ✅ |

## II. Настройки планировщика (на пределе эффективности)

| Параметр | Рекомендация | Почему |
|---|---|---|
| **Целевое удержание** | **90%** (критичные темы до 92%); при перегрузе временно 85% | «Правило 85%» (Wilson 2019); 90→95% ≈ **×2** повторений, 90→97% ≈ ×4 ✅ |
| **learningSteps** | `[10м, 1ч]`; для сложных концептов `[10м, 1д]` | Внутридневной шаг + «ночь сна» перед выпуском |
| **graduatingInterval** | 1 день, easy 3–4 дня | Цепочка Good: 1→3→8→20→50 дн; до зрелости ~32 дня |
| **Новые/день** | 15–20, держать неделями. **0 в последние 25–30% срока** | 50/день → выгорание за ~2 недели; новые в конце не успеют созреть |
| **Лимит повторений** | Практически без лимита; нагрузку резать только новыми | Пропущенный due сдвигает за оптимум; backlog не рассосётся |
| **Кнопка Hard** | Бинарно: вспомнил → Good, нет → Again. Hard ≤10–15% | «Ease hell»: Hard −15 п.п. ease без восстановления (поле Hard 120%) ✅ |
| **Порядок сессии** | Сначала ВСЕ повторения, потом новые | Повторения привязаны к кривой забывания; новые откладываются бесплатно |
| **Личи (4–5 lapses)** | Переписать/разбить/мнемоника, не долбить волей | Планировщик не чинит плохую карту |

> В Anki2 спринт-колоды уже настроены на 30 новых/день, 300 повт/день (можно
> снизить новые до 15–20 для устойчивого темпа на длинной дистанции).

## III. Пределы и физиология

| Предел | Значение | Источник |
|---|---|---|
| Рабочая память | **4 ± 1 чанк** (не 7±2) | Cowan 2001 ✅ |
| Устойчивый темп новых | **15–20/день** неделями; 50 → выгорание | гайды сообществ ⚠️ |
| Во что выливается | 15 новых ≈ 100–150 повт/день через месяц | множитель ~7–10 повт/карта ⚠️ |
| Время на карту | ~6–12 сек | оценка ⚠️ |
| Spacing | распределённо > массированно в **96%** сравнений | Cepeda 2006 (839 оценок) ✅ |
| Сон | консолидация гиппокамп→неокортекс в медленном сне | Diekelmann & Born 2010 ✅ |
| Учёба перед сном | усиливает удержание фактов | PLoS One 2012 ✅ |
| Время суток | факты лучше днём/во 2-й половине; спад 13–15 ч | обзоры, слабее ⚠️ |

## IV. Ежедневный протокол

1. **Утро (5–15 мин):** только накопившиеся повторения, без новых — память свежа после сна.
2. **День:** добить остаток повторений. Блоки по 20–25 мин с перерывами 5 мин (spacing внутри дня).
3. **2-я половина дня:** новые карты, лимит 15–20, держать неделями.
4. **Атомарно:** 1 карта = 1 факт (уважение к пределу 4±1).
5. **Личи:** переписывать или suspend, не долбить.
6. **Вечер (последние 1–2 ч):** лёгкий проход важного/нового — усиливает ночную консолидацию.
7. **Сон 7–9 ч** — обязательная фаза протокола, а не «после».
8. **Раз в неделю:** смотреть фактическое удержание и прогноз нагрузки (вкладка «Статистика»).
9. **Нагрузкой управлять через удержание**, а не лимит повторений: перегруз → временно 85%.

### Поправки скептика (важно)
- **Hard interval = 120%** (не 130%).
- **Аэробика сразу после учёбы для фактов — сомнительна** ⚠️: ведущее исследование
  (van Dongen 2016) показало эффект при нагрузке **через ~4 часа**, а сразу после —
  нет. Оригинальный источник (Thomas 2016) — про *моторную* память.
- Цена удержания: верное число — **90→95% ≈ ×2** повторений (не +35%).

## V. Программистские тренажёры (раздел «Тренажёры»)

В духе Wikium/NeuroNation, но на коде — качают навыки, которые SRS не тренирует:

| Тренажёр | Навык | Что развивает |
|---|---|---|
| Таблица Шульте (числа→hex→бинарные) | внимание | скорость восприятия, периферийное зрение |
| Код N-назад | рабочая память | удержание 4±1 в активном буфере |
| Найди баг | скорость | мгновенное распознавание ошибок (код-ревью) |
| Ментальный интерпретатор | логика | вычисление выражений в уме |

Сложность адаптивная: точность ≥85% → уровень вверх, <60% → вниз («правило 85%»).
